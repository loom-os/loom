# Contributing to Loom

Thank you for your interest in contributing to Loom!

## How to Contribute

### Reporting Bugs

Create an issue on [GitHub Issues](https://github.com/loom-os/loom/issues) with:

- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Environment info (OS, Rust version, etc.)

### Feature Requests

Create an issue tagged `enhancement` describing:

- Use case
- Expected behavior
- Possible implementation (optional)

### Code Contributions

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-feature`
3. Write code following Rust conventions
4. Add tests and update documentation
5. Run: `cargo test && cargo clippy && cargo fmt`
6. Commit: `git commit -m "feat: your feature"`
7. Push and create a pull request

**Commit Format:**

- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation
- `refactor:` Code refactoring
- `test:` Tests
- `chore:` Build/tools

## Development Guidelines

- Use `rustfmt` for code formatting
- Follow Rust API Guidelines
- Write clear comments in English
- Include unit and integration tests
- Update documentation for public APIs

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Focus on technical discussions
